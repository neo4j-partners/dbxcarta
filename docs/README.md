# Documentation

The docs tree is organized by purpose:

- `assets/` — generated or editable visual assets used by docs.
- `explanation/` — conceptual background and design rationale.
- `proposal/` — active implementation proposals only.
- `reference/` — stable technical references for the current codebase.
- `schemapile/` — SchemaPile example notes and generated context artifacts.
- `security/` — supply-chain and security reference material.
